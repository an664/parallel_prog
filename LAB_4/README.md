# Лабораторная работа №4. CUDA

## Задание

Модифицировать программу из лабораторной работы №1 для параллельной работы по
технологии CUDA. Провести эксперименты с разными размерами матриц и различными
конфигурациями сетки блоков.

## Реализация

CUDA-программа находится в [matrix_cuda.cu](./matrix_cuda.cu). Реализованы два
варианта ядра:

- `naive`: каждый CUDA-поток вычисляет один элемент результирующей матрицы;
- `tiled`: блок потоков загружает фрагменты матриц в shared memory и затем
  вычисляет элементы результата.

Размер блока задается параметром запуска программы. Сетка блоков выбирается
автоматически так, чтобы покрыть всю матрицу:

```cpp
dim3 block(block_size, block_size);
dim3 grid((n + block.x - 1) / block.x, (n + block.y - 1) / block.y);
```

## Запуск на CUDA-хосте

Для выполнения нужен NVIDIA GPU и установленный CUDA Toolkit с `nvcc`.

```bash
make
./matrix_cuda sample_A.txt sample_B.txt result.txt 16 tiled
python3 verify.py sample_A.txt sample_B.txt result.txt
```

Полный эксперимент:

```bash
python3 benchmark.py
python3 plot_results.py
```

По умолчанию benchmark запускает размеры `256, 512, 1024, 1600, 2000`, размеры
блоков `8, 16, 32` и режимы `naive, tiled`. Результаты должны попасть в
`results.csv`, а график в `time_vs_cuda_config.png`.

## Выводы

Код лабораторной работы подготовлен, но экспериментальная часть требует
CUDA-совместимую машину. До запуска на NVIDIA GPU корректный вывод о времени,
ускорении и лучшей конфигурации блока сделать нельзя. Ожидаемо режим `tiled`
должен быть быстрее `naive` на больших матрицах за счет уменьшения числа
обращений к глобальной памяти, но это должно быть подтверждено фактическими
замерами на целевом GPU.
